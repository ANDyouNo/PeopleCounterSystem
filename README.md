# People Counter System v2.0

Система подсчёта людей в реальном времени по видео с камеры.
YOLOv8 детекция + ByteTrack трекинг + debounce-защита от дребезга.
Результаты записываются в SQLite базу данных для последующего анализа.
Оптимизировано для работы на слабых CPU (Intel i3 и подобных).

## Быстрый старт

### 1. Установка

```bash
pip install -r requirements.txt
```

### 2. Запуск

```bash
python main.py
```

При первом запуске:
- Настройте зоны исключения (если нужно)
- Модель YOLOv8n скачается автоматически (~6 MB)

### 3. Ускорение через ONNX (рекомендуется для Intel CPU)

```bash
python scripts/export_model.py   # Экспорт модели, один раз
python main.py            # Автоматически подхватит ONNX
```

## Принцип работы

Система определяет количество людей в кадре с помощью YOLOv8 и отслеживает их через ByteTrack. Если человек попал в поле зрения камеры — он считается. Debounce-защита предотвращает ложные срабатывания: если детекция пропала на 2-4 кадра (настраивается), человек не считается ушедшим.

Все данные о присутствии записываются в SQLite базу (`people_counter.db`): интервалы присутствия, максимальное/среднее количество людей, сырые данные детекции.

## Настройка производительности

Все параметры в `config.py`:

| Параметр | Описание | По умолчанию |
|---|---|---|
| `INFERENCE_SIZE` | Размер входа модели (меньше=быстрее) | `320` |
| `SKIP_FRAMES` | Детекция каждый N кадр (больше=быстрее) | `2` |
| `FRAME_WIDTH/HEIGHT` | Разрешение камеры | `640x480` |
| `INFERENCE_BACKEND` | `auto` / `onnx` / `pytorch` | `auto` |
| `CONFIDENCE_THRESHOLD` | Порог уверенности детекции | `0.4` |
| `CAMERA_INDEX` | `None`=выбор при запуске, число=конкретная камера | `None` |
| `DEBOUNCE_FRAMES` | Кадров без детекции до "человек ушёл" | `4` |

### Рекомендации по производительности

**Intel i3 (старые):** `INFERENCE_SIZE=320`, `SKIP_FRAMES=3`, обязательно ONNX

**Intel i5/i7:** `INFERENCE_SIZE=320`, `SKIP_FRAMES=2`, ONNX опционально

**Apple M1+:** `INFERENCE_SIZE=640`, `SKIP_FRAMES=1`, PyTorch достаточно

## База данных

Система записывает данные в SQLite (`people_counter.db`). Подробная документация схемы, SQL-запросы для фронтенда и рекомендуемые REST API эндпоинты — в файле `DB_SCHEMA.md`.

Основные таблицы:
- `presence_log` — интервалы присутствия (вошёл → ушёл)
- `detections_raw` — сырые данные детекции (каждые N кадров)

## Зоны исключения

Зоны исключения позволяют игнорировать области кадра (окна, зеркала и т.п.):
- При первом запуске система предложит настроить зоны
- Нарисуйте прямоугольники drag-and-drop в окне
- Зоны сохраняются в `exclusion_zones.json`

## Troubleshooting

**Камера не найдена:** проверьте подключение, попробуйте `--camera 0`, `--camera 1` или настройки камеры в config.py

**Низкий FPS:** увеличьте `SKIP_FRAMES`, уменьшите `INFERENCE_SIZE`, экспортируйте в ONNX

**Ложные срабатывания:** увеличьте `CONFIDENCE_THRESHOLD` до 0.5-0.6, настройте зоны исключения

**Дребезг (человек то появляется, то исчезает):** увеличьте `DEBOUNCE_FRAMES` в config.py


## Работа с внешним контроллером (сценарий - освещение)

Внешний контроллер - ESP8266

При настройке ESP_ENABLED в config.py контроллер автоматически обнаруживается в локальной сети и получает сигналы о включении/выключении освещения в зависимости от наличия людей в помещении

Другие параметры описаны в config.py

**Прошивка для esp8266 нахоится в соответствующей папке**
